Desafio de Teste Prático

O teste consiste na codificação de uma aplicação Web com conexão a um Banco de Dados estruturado (SQL) e com os seguintes requisitos funcionais:

  1. CRUD de Container
    1. Cliente
    2. Número do container (4 letras e 7 números. Ex.:TEST1234567)
    3. Tipo: 20 / 40
    4. Status: Cheio / Vazio
    5. Categoria: Importação / Exportação
  2. CRUD de Movimentação
    1. Tipo de movimentação (Embarque, Descarga, Gate In, Gate Out, reposicionamento, pesagem, scanner)
    2. Data e Hora do início
    3. Data e Hora do fim
  3. Relatório com o total de movimentações agrupadas por cliente e tipo de movimentação
    1. No final do relatório deverá conter um sumário com o total de importação / exportação
  4. Premissa para todos os artefatos
    • O layout dos controles de manutenção dos dados não precisa ser de um CRUD padrão. Pode se sentir à vontade para criar mecanismos de interação mais criativos.
    • Validações
    • Ordenações
    • Filtros
